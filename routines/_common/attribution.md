Every PR or issue this routine creates MUST be self-identifying. Three layers — title suffix, label, body Provenance block. If any layer is missing, the user cannot tell which routine made it.

- **PR title:** `<conventional-prefix>(<scope>): <description> [routine:<basename>]` — `<basename>` is this routine's file basename. No emoji in titles, bodies, commit messages, or release notes; the conventional-commit prefix is preserved so release-please keeps parsing. **Issue title (no conventional prefix):** `[routine:<basename>] <description>`.
- **Body:** ends with a Provenance block (appended; the rest of the body stays whatever this routine writes):

  ```markdown
  ---

  ## Provenance

  - **Generated by:** [<Routine Name>](<PROMPT_SOURCE_URL>) - cloud routine, <cron description>
  - **Triggered:** <what fired this run (cron + task selection if any)>
  - **Why this PR/issue:** <one-line rationale tying the selection algorithm to this output>
  - **State:** <state gist name/link, if this routine keeps one>
  - **Label:** `cloud-routine`
  ```

- **Label:** apply `cloud-routine` after creation. The label is propagated to every public repo by `JacobPEvans/.github` label-sync — never `gh label create` per repo. If a label-add fails because the target repo is private and outside the sync list, log a Slack warning and proceed.

  ```bash
  gh pr edit "$PR_NUMBER" --repo "$GH_OWNER/<repo>" --add-label cloud-routine
  gh issue edit "$ISSUE_NUMBER" --repo "$GH_OWNER/<repo>" --add-label cloud-routine
  ```
